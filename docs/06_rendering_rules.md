# Правила рендеринга

## ASCII

- Матрица выводится в виде двух таблиц: низкий риск и высокий риск.
- В каждой таблице — 2×2 клетки (активность × масштабируемость).
- Диаграммы стрелок показывают типовые пути (A1→A2→P2 и т.д.).

## Markdown

- Таблицы совпадают по структуре с ASCII.
- Включаются примеры из ячеек и списки мостов.
- Итоговый документ `full_summary.md` собирает все разделы.

## Graphviz

- Узлы: A1..P4 с label.
- Рёбра: мосты из bridges.yaml.
- Используется rankdir=LR.
